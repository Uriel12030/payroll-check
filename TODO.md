## תגובות מייל צריכות להגיע כ-reply ולא כמייל חדש

**עדיפות:** גבוהה

### תיאור
כשמגיבים ללקוח מהמערכת (פאנל אדמין), המייל מגיע אליו כמייל חדש לגמרי במקום כתשובה (reply) לשרשור הקיים.

### התנהגות נוכחית
- כל מייל שנשלח מהמערכת נפתח כשרשור חדש בתיבת הדואר של הלקוח
- אין המשכיות בשיחה

### התנהגות רצויה
- מייל תגובה מהמערכת יגיע כ-reply לשרשור המקורי
- הלקוח יראה את כל ההתכתבות בשרשור אחד

### פרטים טכניים
- לשלוח headers מתאימים: `In-Reply-To` ו-`References` עם ה-`Message-ID` של המייל המקורי
- לשמור את ה-`Message-ID` של כל מייל נכנס ב-DB
- בשליחת תגובה, לשלוף את ה-`Message-ID` המקורי ולצרף אותו כ-header

### קבצים רלוונטיים
- `src/lib/email/resend.ts` — שליחת מיילים
- `src/app/api/admin/email/route.ts` — API route לשליחת מייל מהאדמין
- `src/components/admin/email/` — קומפוננטות UI של המייל
- `src/types/email.ts` — טיפוסי מייל
